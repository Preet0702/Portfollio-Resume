Problem

Athens pricing changes a lot around UGA events. Many hosts guess rates or copy nearby listings, which can miss demand spikes or overprice slower weeks. I built a tool that converts local demand signals into day-specific pricing guidance and competitive context.

What I built

  • Data collection: scraped and processed 650+ Airbnb listings via Apify, then cleaned and stored the dataset in SQLite using Python.
  • Feature engineering: created date-level features including weekends, seasonality, UGA football games, and academic calendar events.
  • Modeling: trained a multivariable linear regression model to estimate pricing premiums and discounts tied to university events.
  • Validation: checked model significance and distributions using JMP.
  • Product layer: built an interactive Streamlit dashboard that outputs day-specific pricing recommendations and competitive insights.
